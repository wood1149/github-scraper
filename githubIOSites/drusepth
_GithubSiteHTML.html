Current Projects

There's only so much time to go around. These projects may not be updated every day, but they are always on my mind and fighting for development time. Each are on GitHub, and each would welcome a commit or two with open arms.

PokeAI

PokeAI is a Javascript framework for writing artificial intelligence agents to compete with human players in Pokemon matches on Pokemon Showdown.

In short, it abstracts out the logic of interacting with the website so you are free to focus entirely on decision making and strategy so you can destroy all those eight year olds that are playing.

Source Code | Example Client

Ankov, the Internet Beast

Ankov is a modular entity that is free to roam the Internet, collectively learning and interacting with various communities around the web.

Integrations are written as tentacles which can then visit websites, connect to machines, hop on IRC, or any other means of communication. A shared ngram database built from everything Ankov sees serves as a database for generating responses and pretending to be just another guy on the Internet.

Source Code | Example Tentacle

Real-time entity tracking

This project uses named entity recognition heuristics to detect and track semantic properties on characters and locations in prose as it's written.

For example, writing "Bob is CEO of his business in Michigan" takes note of a character (Bob, who is a male, holds the job title of CEO, and resides in Michigan), and a location (Michigan, containing Bob and Bob's business), and presents contextual cards of information beside the prose with detailed information about the things you are writing about.

Source code and demo coming soon.

Past Projects

I've been making awesome things since I was a wee lad. These are some of my favorites.

Deanonyora: Authorship Detection

This personal research project scrapes and indexes Quora user activity, building individualized linguistic fingerprints for a subset of power users.

These fingerprints can then be compared to a standalone fingerprint generated from any anonymous answer in order to rank which indexed users are most likely to be the man (or woman) behind the anonymous mask, based on metrics such as preposition frequency, reading level, structural decisions, temporal patterns, and many more.

Explanation on Quora.com

Igor, Voice Assistant

Igor is a voice-activated assistant in the same category as Siri and Cortana, except it's built as a web app on top of HTML5's x-webkit-speech feature.

Igor's greatest strength lies in the ease of adding new integrations: in just a handful of lines of Javascript, any company can write and distribute their own integrations, allowing users to interact with their app, website, or service entirely by voice.

Source Code

Stegkov, steganography meets markov

Steganography is the art of hiding messages in plain sight: secretly hidden away in places that don't even look like they'd contain a secret message—unless you knew to look there.

Markov generation is a common albeit naive technique in generating natural language. Stegkov incorporates steganographic measures in the process, allowing for the encoding and decoding of messages within markov chains.

Source Code

IRC.NET

IRC.NET is a C# DLL that can be included in any C# application to instantly provide powerful IRC functionality: connecting to networks, joining channels, listening for text, and responding to events.

Source Code | Example Usage (a game I wrote)

voice2code (Project Bespoke)

Originally named Bespoke, voice2code originated from a desire to be able to code while my hands were busy: playing Starcraft, driving, while writing other code—you know, the usual times you wish your hands could write the code you're thinking.

This proof-of-concept language defines its own easily-spoken syntax and listens for instructions over a voice input device (such as x-webkit-speech) and translates into executable Javascript to run.

Proof of Concept | Example Use | Compiler (WIP)

Panel

Panel is an interesting C# framework that allows you to give the objects in your code opinions about what qualities make other objects more or less desirable.

I'm not entirely sure what this could actually be used for, but it could lead to preferential treatment of other variables based on their state, or even class racism.
